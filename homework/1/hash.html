Hash Tables are a unique kind of combination of search and data structure. Hash tables consist of key/value pairs, with the keys being used to look up their corresponding values.

The algorithm does this by hashing the key to return a hash value. Hashing is a process which takes in any string input, and outputs a long string of specified length.

This string is repeatable, but not reversable, and is also not guaranteed to be unique to each key.

The table hashes each new key and uses the hash value to assign the corresponding value an index.

Because the hashes are likely, but not guaranteed to be unique, the table must have some way to deal with two keys hashing to the same value (called a hash collision.

Hash tables are fast relative to other search options, with the search time remaining constant regardless of the number of items the algorithm has to search through.

Here we see a phone number lookup table in an address book application.

The key John Smith is a string and is it fed into the hashing function.

The function outputs a hash, the value of which is used to find the appropriate index of the table (referred to as buckets)

Requirements

Hash lookup functions are required to be preformed on lookup tables

Hash lookup must have a way to deal with potential hash collisions (one bucket containing the values of two different keys).
